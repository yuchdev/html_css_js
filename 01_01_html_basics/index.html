Basics of the Web

Definitions

HTML, CSS and JS could be considered as three main pillars of the Web. HTML is responsible for structure, CSS for style and layout, and JavaScript for behavior of the web page.

Standardization and Intro into W3C

Most W3C work revolves around the standardization of Web technologies. If the feature is defined in standard, it does not mean it's implemented in actual browsers. Standardization includes multiple documents, defining subsets of the WWW. Standard texts have very "academic" style, and unlikely fit for studying Web technologies. W3C Standards cover such important domains, like:

  • HTML&CSS - fundamental technologies for building Web
  • JavaScript - ECMAScript Standard (more widely known as JavaScript) is developed by ECMA, however, many of the client-side JavaScript APIs are being developed by W3C
  • Graphics - the most well-known graphical formats for Web are PNG and SVG
  • Many more: Audio&Video, Mobile Web, Localization, etc.

HTML Elements

HTML elements defined by constructs called tags. Most tags look like <tag></tag>, where <tag> is called start tag, and </tag> is called end tag. HTML element is everything from the start tag to the end tag.

Tag can't contain front spaces: < tag> - this is wrong!

Tag trailing spaces are ignored: <tag >

Some elements can't have content, and their tags can be "start only", e.g. line break tag <br>, horizontal line tag <hr>. Such elements are called empty elements.

If HTML element can have content, but does not, its start and end tag can be the one, and look like <tag >. Such tags called self-closing.

In HTML5 it is illegal to make self-closing tags if one can open and close them explicitly.

Document structuring

Headings have levels from 1 to 6

Headings tell ONLY about the document structure, not about the style. This is so-called Structural markup

Browsers automatically add some space (margin) before and after each paragraph element.
The margins can be modified with CSS (with the margin properties)
<br> tag is a single line break

The <hr> tag defines a thematic break in an HTML page (e.g. a shift of topic). Also, it is used to separate content (or define a change) in an HTML page.

Since HTML4, tags could be block-level or inline.

A block-level element always starts on a new line. A block-level element always takes up the full width available (stretches out to the left and right as far as it can). A block level element has a top and a bottom margin, whereas an inline element does not. Ex. The <div> element is a block-level element.

An inline element does not start on a new line. An inline element only takes up as much width as necessary. Ex. The <span> element is an inline element.

Root document index.html

The main homepage of a site written in HTML (and the homepages of each section in a child folder) is called index.html. Web servers are usually set up to return the index.html file if no file name is specified.

HTML categories

HTML5 introduces 9 HTML categories:

  • Metadata content includes elements for marking up document metadata; marking up or linking to resources that describe the behaviour or presentation of the document; or indicate relationships with other documents
  • Flow content. Most of the elements used to mark up the main content in the body of a page are considered to be flow content. In general, this includes elements that are presented visually as either block level or inline level.
  • Sectioning root elements can have their own outlines, but the sections and headers inside these elements do not contribute to the outlines of their ancestors.
  • Sectioning content is used for structuring a document into sections, each of which generally has its own heading. These elements provide a scope within which associated headers, footers and contact information apply.
    HTML Navigational Element <nav> defines a section that contains navigation links that appear often on a site.
    HTML Article Element <article> defines a piece of self-contained content.
    HTML Section Element <section> defines a section of a document to indicate a related grouping of semantic meaning.
    HTML Aside Element <aside> defines a section that, though related to the main element, doesn't belong to the main flow, like an explanation box or an advertisement.
  • Heading content includes the elements for marking up headers. Headings, in conjunction with the sectioning elements, are used to describe the structure of the document.
  • Phrasing content includes text and text-level markup. This is similar to the concept of inline level elements in HTML 4.01. Most elements that are categorised as phrasing content can only contain other phrasing content.
  • Embedded content includes elements that load external resources into the document. Such external resources include, for example, images, videos and Flash-based content.
  • Interactive elements are those that allow the user to interact with or activate in some way. Depending on the userâ€™s browser and device, this could be performed using any kind of input device, such as, for example, a mouse, keyboard, touch screen or voice input.
  • Transparent Content Models. Some elements have transparent content models, meaning that their allowed content depends upon the parent element. They may contain any content that their parent element may contain, in addition to any other allowances or exceptions described for the element.

Structural and semantic markup

Structural markup embeds information about the structure of a document. Structure includes elements like headings, paragraphs, breaks, lists, tables. Like this paragraph, for example.

Semantic markup is the use of an HTML language to emphasize information about the meaning of each element in a document through proper choice of markup elements.
