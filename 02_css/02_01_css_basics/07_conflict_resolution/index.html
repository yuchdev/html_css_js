CSS Conflict Resolution

Why CSS is Cascading Stylesheet

CSS is called cascading, because this is exactly how its rules are applied. The simple rule of CSS conflict resolution could be said like: "The last declaration wins". Also, it could be said, that HTML is processed from top to bottom, and CSS is processed from bottom to top.

When CSS rules do not conflict, they are being merged. E.g. this paragraph is styled from the external CSS file to have light-gray background and blueviolet text color. At the same time, it is directly styled to have Arial Mono font. These rules are being merged, because they are not conflicting.

External CSS is applied after the place where is inserted the HTML code. Usually it is placed in the <head> section.

Inheritance in HTML document

Technically HTML document is a tree of elements. Each element has its own style, however, its child elements inherit the style of their parent.

This <p> element is inherited from the <div> element, and it inherits the font, but has its own background color.

Specificity of CSS rules

CSS rules have a specificity, which is a number of the rules that are applied to the element. The more specifically rules are applied, the higher is the specificity. The most specific rule is applied first.

Different ways of applying the rule have different degree of specificity. Let's explain this. Imagine that higher element in the following list offers higher specificity:

  • CSS rule with the !important keyword overrides any rules.
  • style attribute of the element
  • Styling according to id attribute of the element
  • Styling according to class, pseudo-class, or pseudo-element
  • Styling according to the element relationship (child, descendant, sibling, etc)
  • Styling according to the direct element type
