CSS Styling Forms

What exactly you can style

Like pretty much any other element, you can greatly customize the appearance of forms. Size, color, font, padding, margin, border, and so on.
Important to note that you can style the form only of specific type, e.g. input[type=text], input[type=password], input[type=submit], etc.

Forms sizing

Let's start with the sizing. You can set the size of the form elements using width, height, margin, padding, border, border-radius, styling each side individually, etc.

Good practice would be to include all these borders and margins into the size of the form, for what you can set the property box-sizing: border-box.

Color and size styling with pseudo-classes

Form elements style really shines when you start using pseudo-classes, e.g. :focus, :hover, :active, etc. Here we change the background-color and width properties of the form elements, when the form is focused: input[type=text].search-form:focus, and slightly highlight border-color when the mouse is over it: form.login-form:hover.

Note, here we used the icon inside the input: background-image: url('img/searchicon.png'), and added simple smooth animation using the transition: width 0.4s ease-in-out property.

One more usage of pseudo-classes is validation in response to the user's actions.

Styling textarea

There are some specific properties of textarea, e.g. resize can turn on and off the ability to resize the textarea horizontally, vertically or both, overflow responsible for the behavior of the textarea when it is too small to fit the content (the text scroll, becomes hidden or remains visible, spilling out of the box).

Responsive design and forms

Responsive design is a standard practice in a modern web development, and utilized in the form of media queries (see the chapter on media queries), and forms is not an exception.
