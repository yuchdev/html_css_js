JavaScript Functions

JS Functions

Declaring a function in JS is intuitive:

  function myFunction(p1, p2) {
    return p1 * p2;
  }

The function above is called function declaration. It is a function that is bound to an identifier (name). The function can be invoked using operator (). Without it, the function is not executed but rather returns a reference to the function itself. E.g.

  let x = myFunction; // x is a reference to myFunction
  let y = myFunction(); // y is the result of calling myFunction

Function as Objects

Functions are objects, and can be used like any other object. For example, they can be stored in variables, passed as arguments to other functions, and returned from functions. Such functions are called first-class functions, e.g.:

  let x = function (a, b) {return a * b};
  let z = x(4, 3);

Local Variables

Variables declared within a JavaScript function, become local to the function. Local variables have function scope: they can only be accessed from within the function. Local variables are created when a function starts, and deleted when the function is completed. Local variables declared only with let or const keywords. Using var keyword result in visibility of the variable outside the function. E.g.

  function myFunction() {
    let carName = "Volvo";
    var carModel = "XC90";
  }
  myFunction();
  console.log(carName); // will result in error
  console.log(carModel); // will print "XC90"

Arrow Syntax

Arrow functions are a new way to write functions in JavaScript. They are a shorter syntax for writing function expressions. Arrow functions are always anonymous. They cannot be used as constructors and don't have their own this, arguments, super, or new.target. Arrow functions also cannot be used as object methods. E.g.

  let x = (a, b) => a * b;
  let z = x(4, 3);

Arrow functions handle the this object differently than regular functions. In regular functions, the this keyword represented the object that called the function, which could be the window, the document, an object, etc. With arrow functions, the this keyword always represents the object that defined the arrow function. E.g.

  let person = {
    firstName: "John",
    lastName : "Doe",
    fullName : function() {
      return this.firstName + " " + this.lastName;
    }
  };
  console.log(person.fullName()); // will print "John Doe"
  let person2 = {
    firstName: "Mary",
    lastName : "Doe",
    fullName : () => {
      return this.firstName + " " + this.lastName;
    }
  };
  console.log(person2.fullName()); // will print "undefined undefined"

Sometimes regular functions is what you need. E.g. for implementing a constructor function or object methods.
