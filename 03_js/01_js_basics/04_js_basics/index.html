JavaScript Basic Use

JS letiables and Values

There are two types of letiables in JavaScript:

  • Const (fixed) letiables, declared with const
  • letiable values, declared with let (old method) or let (new method)
  const pi = 3.14;
  let x = 10;
  let y = 20;

Note: let should be used only for old browsers support. New scripts should use let (its support was implemented in all major browsers in 2015-2017) There are important differences between them:

  • let is function-scoped, let is block-scoped
  • let can be re-declared, let can't
  • let can be re-assigned, let can't

Note 2: always prefer const over let, when possible. It must be initialized, it can't be re-assigned, can't be re-declared, and it's block-scoped.

JS Statements

JavaScript statements are separated by semicolons. Multiple statements can be written on one line, separated by semicolons. Ending statements with semicolons is optional, but it's recommended. Statements can be grouped in blocks, enclosed in curly brackets.
Example:

  {
    let x = 10;
    let y = 20;
    let z = x + y;
  }

JS Literals

JavaScript literals are fixed values, not letiables. There are several types of literals:

  • String literals, enclosed in double or single quotes, e.g. "Hello" or 'Hello'
  • Numeric literals, including integers and floating-point numbers, e.g. 10 or 3.14
  • Boolean literals, with values true and false
  • Array literals, enclosed in square brackets, e.g. [1, 2, 3]
  • Object literals, enclosed in curly brackets, e.g. {name: "John", age: 30}

JS Operators

There are several types of operators in JavaScript:

  • Arithmetic operators
  • Assignment operators
  • Comparison operators
  • Logical operators
  • Bitwise operators
  • String operators
  • Conditional (ternary) operator
  • Comma operator
  • Unary operators
  • Relational operators

Naming Conventions

There are simple rules of naming letiables and functions:

  • letiable names can contain letters, digits, underscores, and dollar signs.
  • letiable names must begin with a letter
  • letiable names are case sensitive (y and Y are different letiables)
  • Reserved words (like JavaScript keywords) cannot be used as letiable names

JS Arrays

Creating an array

There are several ways to create an array:

  let cars = ["Saab", "Volvo", "BMW"];
  let cars = new Array("Saab", "Volvo", "BMW");
  let cars = new Array(3);
  cars[0] = "Saab";
  cars[1] = "Volvo";
  cars[2] = "BMW";

Note: For simplicity and readability, the first method is preferred.

Facts about JS arrays

  • Arrays are objects
  • Arrays are mutable
  • Arrays are zero-based
  • Arrays are dynamic
  • Arrays are not type-safe
  • Array indexes are always converted numbers. When you try to use string indexes, JS will redeclare array into an object

Adding and removing elements from front and back

Adding and removing elements from the back is called push and pop:

  let fruits = ["Banana", "Orange", "Apple", "Mango"];
  fruits.push("Kiwi");
  fruits.pop();

Adding and removing elements from the front is called unshift and shift:

  let fruits = ["Banana", "Orange", "Apple", "Mango"];
  fruits.unshift("Lemon");
  fruits.shift();

Loops in JavaScript

Like most of the programming languages, JavaScript has several types of loops:

  • for - loops through a block of code a number of times
  • for/in - loops through the properties of an object
  • for/of - loops through the values of an iterable object
  • while - loops through a block of code while a specified condition is true
  • do/while - also loops through a block of code while a specified condition is true

The for Loop

  for (i = 0; i < 5; i++) {
    text += "The number is " + i + "\n";
  }

The for/in Loop

  let person = {fname:"John", lname:"Doe", age:25};
  let text = "";
  let x = "";
  for (x in person) {
    text += person[x];
  }

The for/of Loop

  let cars = ["BMW", "Volvo", "Mini"];
  let text = "";
  let x = "";
  for (x of cars) {
    text += x + "
";
  }

The while Loop

  let text = "";
  let i = 0;
  while (i < 10) {
    text += "The number is " + i;
    i++;
  }

The do/while Loop

  let text = "";
  let i = 0;
  do {
    text += "The number is " + i;
    i++;
  }
  while (i < 10);

Visibility Scopes

Visibility scopes are the areas of the code where a variable is visible. In JavaScript, there are 3 types of scopes, intuitive for those who is a familiar with programming:

  • Global scope
  • Function scope
  • Block scope
  let x = 10; // Global scope
  function myFunction() {
    let y = 20; // Function scope
    {
      let z = 30; // Block scope
    }
  }

However, in older versions of JavaScript, there was only 2 types of scopes: global and function. Variables declared with var keyword were visible in the whole function, not only in the block. Scopes were constantly violated, and it was a big problem. For example:

  var x = 10;
  function myFunction() {
    {
      var x = 30;
    }
    x = 20;
  }

In modern JavaScript, there are ways to avoid this problem. For example, you can use let keyword instead of var.

Strict Mode

Strict mode is a way to write secure JavaScript code. It is a string literal that is added to the top of the script. Strict mode changes previously accepted "bad syntax" into real errors, which makes it easier to write "secure" JavaScript. These are a couple of examples of strict mode:

  "use strict";
  x = 3.14; // This will cause an error because x is not declared
  function x(p1, p2) {};   delete x; // Deleting a function is not allowed in strict mode
  let x = 3.14;   delete x; // Deleting a variable is not allowed in strict mode
