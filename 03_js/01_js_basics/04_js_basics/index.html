JavaScript Basic Use

JS Variables and Values

There are two types of variables in JavaScript:

  • Const (fixed) variables, declared with const
  • Variable values, declared with var (old method) or let (new method)
  const pi = 3.14;
  var x = 10;
  let y = 20;

Note: var should be used only for old browsers support. New scripts should use let (its support was implemented in all major browsers in 2015-2017) There are important differences between them:

  • var is function-scoped, let is block-scoped
  • var can be re-declared, let can't
  • var can be re-assigned, let can't

Note 2: always prefer const over let, when possible. It must be initialized, it can't be re-assigned, can't be re-declared, and it's block-scoped.

JS Statements

JavaScript statements are separated by semicolons. Multiple statements can be written on one line, separated by semicolons. Ending statements with semicolons is optional, but it's recommended. Statements can be grouped in blocks, enclosed in curly brackets.
Example:

  {
    let x = 10;
    let y = 20;
    let z = x + y;
  }

JS Literals

JavaScript literals are fixed values, not variables. There are several types of literals:

  • String literals, enclosed in double or single quotes, e.g. "Hello" or 'Hello'
  • Numeric literals, including integers and floating-point numbers, e.g. 10 or 3.14
  • Boolean literals, with values true and false
  • Array literals, enclosed in square brackets, e.g. [1, 2, 3]
  • Object literals, enclosed in curly brackets, e.g. {name: "John", age: 30}

JS Operators

There are several types of operators in JavaScript:

  • Arithmetic operators
  • Assignment operators
  • Comparison operators
  • Logical operators
  • Bitwise operators
  • String operators
  • Conditional (ternary) operator
  • Comma operator
  • Unary operators
  • Relational operators

JS Functions

Declaring a function in JS is intuitive:

  function myFunction(p1, p2) {
    return p1 * p2;
  }

Functions in JS are objects, so they can be assigned to variables. They can also be passed as arguments to other functions, and can be returned by other functions. Operator ()) used to call a function, from the code, or from an event handler. Also, there are self-invoking functions, which are invoked (started) automatically, without being called.

Naming Conventions

There are simple rules of naming variables and functions:

  • Variable names can contain letters, digits, underscores, and dollar signs.
  • Variable names must begin with a letter
  • Variable names are case sensitive (y and Y are different variables)
  • Reserved words (like JavaScript keywords) cannot be used as variable names

JS Objects

Object in JavaScript is a collection of named values (JSON dictionary), e.g.

person = {firstName:"John", lastName:"Doe", age:50, eyeColor:"blue"}

Facts about JS objects

  • Declare objects const when possible, let when required
  • Assigning the object to variable, you actually assign the reference to the object
  • This means, the object itself is mutable, even if the variable is declared const, objects are not actually copied in memory, only the reference is copied

There are two ways to access the properties of an object

  • objectName.propertyName
  • objectName["propertyName"]

Methods and keyword this

Objects may contain methods, which are just functions, stored as object properties. Inside the method, you can access the object properties with this keyword, like in many OOP languages. In a global function, this refers to the global object.

Keyword new

The new keyword is used to create an instance of a user-defined object type or of one of the built-in object types that has a constructor function. The new keyword even define basic types like String or Number as objects.

JS Arrays

Creating an array

There are several ways to create an array:

  var cars = ["Saab", "Volvo", "BMW"];
  var cars = new Array("Saab", "Volvo", "BMW");
  var cars = new Array(3);
  cars[0] = "Saab";
  cars[1] = "Volvo";
  cars[2] = "BMW";

Note: For simplicity and readability, the first method is preferred.

Facts about JS arrays

  • Arrays are objects
  • Arrays are mutable
  • Arrays are zero-based
  • Arrays are dynamic
  • Arrays are not type-safe
  • Array indexes are always converted numbers. When you try to use string indexes, JS will redeclare array into an object

Adding and removing elements from front and back

Adding and removing elements from the back is called push and pop:

  var fruits = ["Banana", "Orange", "Apple", "Mango"];
  fruits.push("Kiwi");
  fruits.pop();

Adding and removing elements from the front is called unshift and shift:

  var fruits = ["Banana", "Orange", "Apple", "Mango"];
  fruits.unshift("Lemon");
  fruits.shift();
