Forms in HTML5

Traditionally, the term 'form' has referred to a printed document that contains spaces for you to fill in information. HTML borrows the concept of a form to refer to different elements that allow you to collect information from visitors to your site.

Whether you are adding a simple search box to your website or you need to create more complicated applications, HTML5 forms give you a set of elements to collect data from your users.

  • Adding text
    • Simple text
    • Password
    • Multiline text (text area)
    • Date input
    • Email and URL input
    • Search input
  • Making choices
    • Radio buttons
    • Checkboxes
    • Comboboxes
    • Multiple select boxes
  • Submitting data
    • Submit buttons
    • Image buttons
    • Upload file forms
  • Formatting and validation
    • Group box
    • Labels
    • Basic form validation

Text Forms

Form definition

The best known form on the web is probably the search box that sits right in the middle of Google's homepage.
The <form> is a tag used to create an HTML form in browser for custom user input.

Any form controls live inside a <form> element.
The <form> element can contain one or more of the following form elements:
  • <input>
  • <textarea>
  • <button>
  • <select>
  • <option>
  • <optgroup>
  • <fieldset>
  • <label>
  • <output>

The <form> element should always carry the action attribute, and also usually have method and idattributes.

action

Action attribute provides value of the URL page on the server, that will receive the form information, when it's submitted

method

Method attribute may carry one of two values, get or post. You can read details about both of them at W3School HTTP Reference Here are some reasons to use one of them:
  • GET is good for short and unimportant pieces of data, as it has length restriction
  • GET is cached and saved in browser history, don't use it for any sensitive data
  • POST has no limits on length, it could be used to send large pieces of data, like file
  • POST does not leave traces in cache and browser history, use it for sensitive and private data

id

id attribute is used for unique identification of HTML element on the page, to access it from CSS stylesheet or JavaScript.

Other attributes

The <form> tag has several other useful attributes. autocomplete turns on and off automatic form completion, enctype specifies how the form data should be encoded while submitting it to the server, novalidate turns off form validation.

Input types text and password

Single-line text input used for a single line of text such as logins, email addresses and names.
Password input is like a single line text box but it masks the characters entered.

Type text or password are specific for single-line input forms, name is a server-side form identifier, maxlength is a very basic validation restricting the number of characters to enter.

Even if the password is hidden on the screen, this does not mean that the data in a password control is sent securely to the server. For full security, the server needs to be set up to communicate with users' browsers using Secure Sockets Layer (SSL) and HTTPS protocol.

Tag input attributes

  • accept is a specific the for file input dialog box (only for type="file"). It specifies a filter for file extension and media type
  • alt specifies an alternate text for images (only for type="image")
  • autocomplete specifies if <input> element have autocomplete enabled
  • autofocus specifies if <input> element should automatically get focus when the page loads
  • checked specifies that an <input> element should be pre-selected when the page loads (for type="checkbox" or type="radio")
  • disabled specifies that an <input> element should be disabled
  • form specifies the form ID the <input> element belongs to
  • height and width specifies size if type="image" only
  • list refers to a <datalist> element ID that contains pre-defined options for an <input> element
  • maxlength specifies the maximum number of characters allowed in <input> element, specific for text-based elements only
  • min specifies minimum number or date
  • multiple specifies that a user can enter more than one value in <input> element
  • pattern specifies a regular expression for the <input> element validation
  • placeholder is a short text hint that describes the expected value
  • readonly specifies if input field is read-only
  • required specifies that an input field must have a value before submitting the form
  • src specifies the URL of the image to use as a submit button (only for type="image")

Multiline text input - textarea

The <textarea> element is used to create a mutliline text input. Unlike other input elements this is not an empty element, it should therefore have an opening and a closing tag. Any text that appears between the opening <textarea> and closing </textarea> tags will appear in the text box when the page loads.

If the user does not delete any text between these tags, this message will be sent to the server. It's a common practice to JavaScript to clear this information when the user clicks in the text area.

Dropdown list

The <select> element is used to create a drop down list box. It contains two or more <option> elements.

Dropdown list provides the same functionality as radio buttons. Consider two factors designing your UI

  1. If users need to see all options at a glance, radio buttons are better suited
  2. If there is a long list of options (like a list of countries), drop down list boxes work bette
