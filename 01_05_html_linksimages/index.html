Demonstration of HTML5 links

Link types

You can meet the following types of links

  • Links from one website to another
  • Links from one page to another within the same website
  • Links within the same page
  • Links that open in a new browser window
  • Links that open a email application

Hyperlinks

The <a> tag defines a hyperlink, which is used to link from one page to another. The most important attribute of the <a> element is the href attribute, which indicates the link's destination. The text within <a> tag is known as the link text.

When you link to a different website, the value of the href will be the full web address, also known as an absolute URL. A local link (a link to a page within the same website) is specified with a relative URL (without the "https://www" part). If all the pages of the site are in the same folder, then the the href attribute is just the name of the file. If you use folder name without the name of a file, in this case the web server will try to show the index.html page. A forward slash in the relative URL will link to the homepage for the entire site, and a forward slash followed by a file name will return that file in the root directory.

If the <a> tag has no href attribute, it is only a placeholder for a hyperlink. A linked page is normally displayed in the current browser window, unless you specify another target.

Link appearance

By default, links will appear as follows in all browsers:

  • An unvisited link is underlined and blue
  • A visited link is underlined and purple
  • An active link is underlined and red

Link attributes

We already mentioned href attribute. Other attributes you may find useful

  • download attribute specifies that the target will be downloaded when a user clicks on the hyperlink
  • hreflang specifies the language of the linked document
  • media specifies what media/device the linked document is optimized for. This attribute is used to specify that the target URL is designed for special devices (like handheld, projectors, print previews), speech or print media.
    Possible values:
    • all (default)
    • aural - Speech synthesizers
    • braille - Braille feedback devices
    • handheld - Handheld devices (small screen, limited bandwidth)
    • projection - Projectors
    • print - Print preview
    • screen - Computer screen
    • tty - Teletypes and media using a fixed-pitch character grid
    • tv - Television type devices (low resolution, limited scroll ability)
  • ping attribute specifies a space-separated list of URLs to be notified if the user follows the hyperlink. When the user clicks on the hyperlink, the ping attribute will send a short HTTP POST request to the specified URL. This attribute is useful for monitoring/tracking.
  • referrerpolicy attribute specifies which referrer information to send when the user clicks on the hyperlink.
    Possible values:
    • no-referrer - No referrer information is sent
    • no-referrer-when-downgrade - Default. Sends the origin, path, and query string if the protocol security level stays the same or is higher (HTTP to HTTP, HTTPS to HTTPS, HTTP to HTTPS is ok). Sends nothing to less secure level (HTTPS to HTTP is not ok)
    • origin - Sends the origin (scheme, host, and port) of the document
    • origin-when-cross-origin - Sends the origin of the document for cross-origin request. Sends the origin, path, and query string for same-origin request
    • same-origin - Sends a referrer for same-origin request. Sends no referrer for cross-origin request
    • strict-origin-when-cross-origin - Sends the origin if the protocol security level stays the same or is higher (HTTP to HTTP, HTTPS to HTTPS, and HTTP to HTTPS is ok). Sends nothing to less secure level (HTTPS to HTTP)
    • unsafe-url - Sends the origin, path, and query string regardless of security. Use this value carefully!
  • rel attribute specifies the relationship between the current document and the linked document (author, bookmark, license, help, etc.). Only used if the href attribute is present. Search engines can use this attribute to get more information about a link
  • target attribute specifies where to open the linked document. Value target="_blank" is important to open the linked document in a new browser tab.
    Possible values:
    • _blank - Opens the linked document in a new window or tab
    • _self - Default value. Opens the linked document in the same frame
    • _parent - Opens the linked document in the parent frame
    • _top - Opens the linked document in the full body of the window
  • type attribute specifies the Internet media type (formerly known as MIME type) of the linked document. This attribute is only used if the href attribute is set. Look at IANA Media Types for a complete list of standard media types.
    This attribute is purely advisory.

Bookmarks

Bookmarks are links to the same page, so that reader can jump to specific parts of the page. Bookmarks can be useful if a web page is very long. To create a bookmark - first create the bookmark, then add a link to it. When the link is clicked, the page will scroll down or up to the location with the bookmark.

<link> tag

The <link> tag defines the relationship between the current document and an external resource. The <link> tag is most often used to link to external style sheets. If the <link> element is an empty element, it contains attributes only. Example:
<link rel="stylesheet" href="styles.css">

Some attributes of the <link> tag are being shared with <a> tag:

  • href
  • hreflang
  • media
  • referrerpolicy
  • rel
  • type

However, <link> tag has its specific attributes as well:

  • crossorigin - Specifies how the element handles cross-origin requests
    • anonymous
    • use-credentials
  • sizes - Specifies the size of the linked resource. Only for rel="icon"
  • title - Defines a preferred or an alternate stylesheet

<nav> tag

The <nav> tag defines a set of navigation links. The <nav> element is intended only for major block of navigation links. Browsers, such as screen readers for disabled users, can use this element to determine whether to omit the initial rendering of this content.

HTML5 and images

Image tag

Images can improve the design and the appearance of a web page. We use <img> tag to embed an image into a web page. Technically, images are not inserted into a web page; images are linked to web pages. The <img> tag creates a holding space for the referenced image.

If the <img> tag is empty, it contains attributes only, and does not have a closing tag. It has two required attributes: src and alt.

  • src - Specifies the path to the image
  • alt - Specifies an alternate text for the image

Make sure that the image actually stays in the same spot in relation to the web page, otherwise your visitors will get a broken link icon. The broken link icon and the alt text are shown if the browser cannot find the image.

Image optional attributes

  • crossorigin allows images from third-party sites that allow cross-origin access to be used with canvas; possible values are anonymous and use-credentials
  • width, height are width and height in pixels. Recommended to reserve space on the page, in case of the image has been replaced with broken link icon, to preserve the size.
  • ismap is a boolean attribute. When present, it specifies that the image is part of a server-side image map (an image map is an image with clickable areas). When clicking on a server-side image map, the click coordinates are sent to the server as a URL query string. The ismap attribute is allowed only if the <img> element is a descendant of an <a> element with a valid href attribute.
  • loading attribute specifies whether a browser should load an image immediately or to defer loading of off-screen images until for example the user scrolls near them. Default value is loading="eager". Add loading="lazy" only to images which are positioned below the fold.
  • longdesc attribute specifies a hyperlink to a detailed description of an image.
    Possible values:
    • An id to another element
    • An absolute URL (longdesc="https://www.example.com/description.txt")
    • A relative URL (longdesc="description.txt")
  • referrerpolicy attribute specifies which referrer information to use when fetching an image. Possible values are the same as <a> tag.
  • sizes attribute specifies image sizes for different page layouts
  • srcset is a list of URLs which specifies a list of images to use in different situations
  • usemap attribute specifies an image as a client-side image map (an image map is an image with clickable areas). The usemap attribute is associated with a <map> element's name attribute, and creates a relationship between the <img> and the <map>.
    Note: The usemap attribute cannot be used if the <img> element is a descendant of an <a> or <button> element.

Image Maps

The HTML <map> tag defines an image map. An image map is an image with clickable areas. The areas are defined with one or more <area> tags, that defines the clickable areas in the image map. The required name attribute of the <map> element is usemap attribute and creates a relationship between the image and the map. Server-side image map uses click coordinates, which are being sent with the HTTP request. Client-side image map operates with images and web-pages already existing on the client-side.

Background Images

A background image can be specified for almost any HTML element.
To add a background image on an HTML element, use the HTML style attribute and the CSS background-image property.

You can also specify the background image in the <style> element, in the <head> section.

If the background image is smaller than the element, the image will repeat itself, horizontally and vertically, until it reaches the end of the element. To avoid the background image from repeating itself, set the background-repeat property to no-repeat.

If you want the background image to cover the entire element, you can set the background-size property to cover.
Also, to make sure the entire element is always covered, set the background-attachment property to fixed.
If you want the background image to stretch to fit the entire element, you can set the background-size property to 100% 100%.

Canvas

The <canvas> tag is used to draw graphics, on the fly, using scripting languages, like JavaScript. For proper work, user's browser should have JavaScript enabled.
The tag is transparent, and is only a container for graphics, you must use a script to actually draw the graphics.

The <canvas> tag has two attributes:

  • height specifies the height of the canvas in pixels. Default value is 150
  • width specifies the width of the canvas in pixels. Default value is 300
This text will appear if the browser does not support the canvas tag.

Figure

The <figure> tag specifies self-contained content, like illustrations, diagrams, photos, code listings, etc. While the content of the <figure> element is related to the main flow, its position is independent of the main flow, and if removed it should not affect the flow of the document. The <figcaption> element is used to add a caption for the <figure> element.

Picture

The <picture> element allows you to display different pictures for different devices or screen sizes. It contains one or more <source> elements, each referring to different images through the srcset attribute. This way the browser can choose the image that best fits the current view and/or device.
Each <source> element has a media attribute that defines when the image is the most suitable.
The browser will use the first <source> element with matching attribute values, and ignore any following <source> elements. This behavior is similar to <video> and <audio> elements.

Resize the browser to see different versions of the picture loading at different viewport sizes. The browser looks for the first source element where the media query matches the user's current viewport width, and fetches the image specified in the srcset attribute.

The <img> element is required as the last child tag of the picture declaration block. The <img> element is used to provide backward compatibility for browsers that do not support the picture element, or if none of the source tags matched.
There are the following use cases for the <picture> element:

  1. The most common use of the <picture> element will be for art direction in responsive designs. Instead of having one image that is scaled up or down based on the viewport width, multiple images can be designed to more nicely fill the browser viewport.
  2. Bandwidth. If you have a small screen or device, it is not necessary to load a large image file.
  3. Format Support. Some browsers or devices may not support all image formats. By using the <picture> element, you can add images of all formats, and the browser will use the first format it recognizes, and ignore any of the following elements.
